The pack

Vendor it, and every agent equips a persona.

personas is the gas-city pack behind blackrim-personas: the registry, the equip and match engine, the warm-cache lifecycle, a SessionStart equip hook, and a CLI. Pure standard library, no runtime third-party deps, no prompt fragments to maintain by hand.

Read the pack source See the roster

What it provides

A small, runnable engine.

The pack is the runnable companion to the persona system. Nothing in the decision path is hard-coded: the roster and the warm-cache policy are config, and the engine is a thin, testable core the CLI and the equip hook both call.

PathRole
bin/personasThe list, show, lint, match, equip, cache, evict, sweep CLI.
personas/The engine: registry loader, match rule, warm-cache lifecycle.
personas.tomlThe roster of 10 personas and the warm-cache TTL policy.
hooks/The SessionStart equip-on-task-pickup hook.
docs/DESIGN.mdThe registry shape, the match rule, and the TTL reasoning.

Stdlib-only, on tomllib with a tomli fallback for older Python. It mirrors the minimal footprint of model-advisor and provider-forge.

quickstart
# the roster, and a registry integrity check
personas list
personas lint

# preview the equip decision for a task
personas match "a paginated REST endpoint"

# equip it into the shared warm cache
personas equip --from-bead pers-3f1

list, show, lint, match, and cache are pure reads. equip, evict, and sweep mutate the shared warm cache.

The CLI

One command group per engine layer.

The CLI is the operator surface over the three layers, and adds no logic of its own. Every command is a thin render over the registry, the match rule, or the cache, so the CLI and the equip hook always make the same decision.

LayerCommandsWhat they do
Registry list show lint The roster; one persona's full playbook and bar; an integrity check that gates on edits.
Equip decision match equip Preview the decision read-only, or materialize it into the shared warm cache.
Warm cache cache evict sweep What is warm and how long it has left; drop one persona; age out expired entries.

Install into a city

Vendor it, register it, reload.

Drop pack/ into the target city as packs/personas/, then install at town or rig scope. It registers a direct source = "packs/personas" import, runs gc reload so the equip hook is merged into projected settings, and verifies.

packs/personas
# city-wide
packs/personas/install.sh --town

# one rig
packs/personas/install.sh --rig <name>

# preview without writing
packs/personas/install.sh --town --dry-run

# reverse it (--purge to drop the venv)
packs/personas/uninstall.sh --town
